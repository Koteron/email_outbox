# Outbox Email Mock Sender

Проект реализует паттерн **Outbox** для имитации отправки email-сообщений через фоновых воркеров.

Основная идея:
- записи сначала импортируются в таблицу outbox
- затем отдельные воркеры последовательно обрабатывают их и “отправляют”

## Требования

- Docker + Docker Compose
- Python 3.10+
- UV (менеджер зависимостей)
- Проект использует `pyproject.toml`

---

## Быстрый старт

### 1. Установить UV

```
pip install uv
```

---

### 2. Установить зависимости и создать виртуальное окружение

```
uv sync --frozen --all-extras
```

Что делает команда:
- создаёт виртуальное окружение (обычно `.venv`)
- устанавливает все зависимости из lock-файла
- гарантирует воспроизводимую установку (`--frozen`)

---

### 3. Активировать виртуальное окружение

**Windows:**
```
.venv\Scripts\activate
```

**Linux / macOS:**
```
source .venv/bin/activate
```

---

### 4. Поднять базу данных

```
docker compose up -d
```

Запустит PostgreSQL, необходимый для работы проекта.

---

### 5. Применить миграции

```
python manage.py migrate
```

Создаёт таблицы в базе данных.

---

### 6. Импорт записей в outbox

```
python manage.py import_records --file_path ./input.xlsx --batch_size N
```

Параметры:
- `--file_path` — путь к Excel-файлу
- `--batch_size` — размер батча при записи

---

### 7. Запуск воркеров

```
python manage.py start_sender --scale N --batch_size M
```

Параметры:
- `--scale N` — количество воркеров
- `--batch_size M` — сколько записей обрабатывает каждый воркер за цикл

Поведение:
- воркеры читают записи со статусом `PENDING`
- обрабатывают их
- вызывают mock-отправку email
- обновляют статус на `SENT`

## Архитектура

- **OutboxRecord**
  - модель сообщений
  - поля:
    - `external_id`
    - `user_id`
    - `email`
    - `subject`
    - `message`
    - `status`
    - `sent_at`

- **Import Service**
  - загрузка данных из XLSX
  - батчевая запись в БД

- **Worker Service**
  - обработка сообщений батчами
  - вызов `send_email` (mock)
  - обновление статусов


## Особенности

- Batch processing для оптимизации работы с БД
- Уникальность записей по `external_id`
- Поддержка параллельных воркеров
- Разделение ответственности между импортом и обработкой
- Mock-отправка вместо реального SMTP

## Формат входного файла

Файл `input.xlsx` должен содержать:

- external_id
- user_id
- email
- subject
- message



## Результат работы

После запуска:

1. Данные импортируются в outbox
2. Воркеры начинают обработку
3. Каждая запись получает статус: `SENT`
4. Фиксируется время отправки (`sent_at`)
